# 五个改变效率的 Shell 习惯

在 t2c 里输入命令浏览内容，本质上和日常写代码时在终端里工作是一样的。下面五个习惯，无论用什么 shell，都值得一试。

## 1. 用历史搜索代替重复输入

`Ctrl+R` 反向搜索历史命令。比方向键翻找快一个数量级。在 t2c 里，上下方向键同样遍历命令历史——`grep astro` 跑过一遍，下次直接 ↑ 即可。

## 2. Tab 补全不是可选功能

路径补全、命令名补全、参数补全都值得信任。在 t2c 中输入 `cd doc` + Tab 会补全为 `docs/`。养成「打到唯一前缀就 Tab」的习惯，错误率会明显下降。

## 3. 组合命令，而不是重复劳动

```bash
# 找最近修改的 Go 文件
find . -name '*.go' -mtime -1

# 统计某目录代码行数
wc -l $(find src -name '*.ts')
```

管道和子命令是 shell 的核心优势。一次写对，可以放进 alias 或脚本。

## 4. 用 `less` / `vi` 阅读长输出

`cat` 适合短内容；长日志、长文档用分页器。t2c 的 `vi` 命令模拟了这个习惯——读长文时 `:q` 退出，回到 shell 上下文。

## 5. 保持 cwd 的语义清晰

`pwd` 随时确认当前位置。在 t2c 里 `cd` 改变虚拟工作目录，影响 `ll` 和相对路径的 `cat`。这和真实文件系统一样：**上下文错了，后续命令全错**。

---

这些习惯没有一条需要特殊工具。t2c 只是把其中一部分搬到了浏览器里——如果你在这里用得顺手，真实终端也不会陌生。
