# 终端是一种被低估的界面

大多数网站都在争夺你的注意力：导航栏、侧边栏、推荐流、弹窗 cookie 横幅。终端界面几乎什么都没有——只有一行提示符，等待你输入意图。

## 命令即导航

在 t2c 里，`cd docs` 和 `cat react-hooks.md` 做的事情，和传统网站里的点击文件夹、打开文章没有本质区别。区别在于**肌肉记忆**：如果你已经在终端里工作，切换到这个博客不需要重新学习一套交互模型。

| 传统网页 | t2c 终端 |
|---------|---------|
| 点击目录树 | `cd docs/frontend` |
| 搜索框 | `grep hooks` |
| 阅读文章 | `cat react-hooks.md` 或 `vi react-hooks.md` |
| 按时间浏览 | `timeline` |

## 适合谁

- 已经习惯 shell 的开发者
- 想要**低干扰**阅读体验的人
- 把博客当作**可检索知识库**而非社交媒体的作者

## 不适合谁

如果你从不使用终端，学习成本确实存在。这也是为什么 t2c 保留了 `help` 命令和 Tab 补全——降低第一次使用的摩擦，而不是假设每个人都是资深用户。

## 设计取舍

我们刻意没有引入 xterm.js 或完整的 shell 仿真。第一版只需要：输入一行、得到输出、继续输入。复杂交互（如 vi 阅读器）是后来按需加入的，而不是一开始就堆功能。

终端界面不是复古噱头，而是一种**约束**：约束迫使你思考用户真正需要什么，而不是页面上还能放什么。
